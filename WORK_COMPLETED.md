### What Was Done

**Fixed** the role consistency comparison in `src/evidence/similarity.py` (lines 495-536):

**Changed**:
- Moved `NullLevel` from FIRST position to AFTER positive signals
- Added `same_role_class` level for improved EM training discrimination
- Expanded plausible transitions: added head↔spouse, sibling↔head (previously only son↔head, daughter↔head)
- Reordered levels to: exact → same-class → transitions → NULL → else

**Expected Impact**: Restore linkage from regression (9%) to baseline (26%)

### Actual Result

❌ **The fix WORSENED the regression**:

| Metric | Before Phase 3 | After Original | After My Fix |

| **Pairs ≥0.65** | — | 8.6% | 0.0% |
| **Pairs <0.45** | — | 55.2% | 56.6% |

### Conclusion

**The Phase 3 role consistency feature is fundamentally flawed.** The feature implementation lowers scores and suppresses linkage rather than improving it. Root causes:

1. **Role comparison too restrictive**: Comparison levels fire too selectively
2. **EM training miscalibrated**: Role signals weighted too negatively
3. **Plausible transitions wrong**: Current transitions don't match real data patterns
4. **NULL handling issue**: Missing roles penalize matches

**Recommendation**: 
- Either REMOVE the role consistency feature entirely (revert to v1.1)
- OR redesign the comparison from scratch with actual data analysis of role patterns

### Key Definitions

**Three-Census Linkage %**:
```
100 × (Linked Recorded Persons) / 3,167
```
- Numerator: `COUNT(DISTINCT recorded_person_id FROM person_recorded_person)`
- Denominator: 3,167 (FIXED — all persons across all three censuses)
- Example: 463 linked / 3,167 total = 14.6% linkage

**Pairwise Person Similarity**:
- Total similarity pairs generated by Splink
- Average score across all pairs
- Score distribution by tier (≥0.65, 0.50-0.65, 0.45-0.50, <0.45)
- Expected pattern: clustering around 0.50 threshold
